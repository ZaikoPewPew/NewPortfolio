# Article widget

Статус: **live** · место: страница кейса (не в bento `registry`).

Вертикальный «барабан» навигации по статье — **колонка имён секций** (без правой полосы с индикаторами). Fixed справа, по центру по вертикали, отступ 16px (`--article-offset-right`). Макет: Figma `articleWidget` (node `1065:1814`).

## Файлы

| Файл | Назначение |
|------|------------|
| `ArticleWidget.astro` | Разметка + bind island |
| `article.styles.css` | Fixed справа, колонка имён |
| `article.animations.css` | Reveal (transform не анимируется — идёт за скроллом) |
| `article.client.ts` | Барабан: scroll-spy, scrub, детенты, раскрытие, клик по секции |

## Устройство

- **Секции** — все `##` (`<h2>`) внутри `[data-case-content-body]`. Если секций < 2 — виджет не показывается.
- **Имена**: активное 12px, соседние 11px, дальние 10px; выровнены вправо, кликабельны. В свёрнутом виде видно ~5 имён рядом с активным (`NAME_WINDOW`) тремя ступенями яркости: активное ярко → ±1 приглушённо → ±2 совсем тускло (`NAME_FADE_PER_SLOT`); при раскрытии — полный список секций. Зона клика больше самого текста (прозрачные паддинги сверху/снизу/слева), фон не рисуется. Вся колонка (`--article-drum-height`) — таргет для раскрытия.
- **Пиннинг у краёв:** на первой секции активное имя прижато к верху с отступом 8px (остальные каскадом вниз), на последней — к низу; в середине активное центрируется.

## Поведение

Два состояния:

- **Свёрнуто (default):** тап по колонке → раскрывает полный список.
- **Раскрыто:** клик по имени → навигация, **меню остаётся открытым**; свернуть можно тапом по пустой области колонки, кликом вне виджета или `Esc`.

- **Scroll-spy:** позиция барабана — непрерывный дробный индекс секции от скролла (линия чтения — 35% вьюпорта). У нижнего края страницы позиция принудительно фиксируется на последней секции (иначе её заголовок не доезжает до линии чтения и она не «загорается»).
- **Плавность:** позиция интерполируется покадрово (lerp к цели, фактор `SMOOTH`); раскрытие/закрытие — тайм-based tween (`EXPAND_DURATION` + `easeOutExpo`). `prefers-reduced-motion` → без сглаживания и мгновенное раскрытие.
- **Скролл контента:** **перетаскивание** колонки вверх/вниз (порог `DRAG_THRESHOLD`, чтобы отличать клик от драга) и **колесо** над виджетом проматывают страницу напрямую; клик по секции → плавный скролл (`prefers-reduced-motion` → instant).
- **Hover:** неактивные (приглушённые) имена секций подсвечиваются до полной непрозрачности.
- **Детент:** лёгкое вибро при пересечении секции во время перетаскивания (`FeedbackBus`, только haptic). Клик — `tap`/`hover` через `data-feedback`.
- **Mobile (< 640px):** скрыт.
- Пере-инициализация на `astro:page-load` (View Transitions): старые слушатели снимаются через `AbortController`, `ResizeObserver` пере-меряет офсеты при изменении высоты тела.

Геометрия (размеры/шаги/сглаживание) — константы в начале `article.client.ts`.
