# Article widget

Статус: **live** · место: страница кейса (не в bento `registry`).

Вертикальный «барабан» навигации по статье. Fixed справа, по центру по вертикали, отступ 16px (`--article-offset-right`). Макет: Figma `articleWidget` (node `1065:1814`).

## Файлы

| Файл | Назначение |
|------|------------|
| `ArticleWidget.astro` | Разметка + bind island |
| `article.styles.css` | Fixed справа, пилюля-барабан, колонка имён |
| `article.animations.css` | Переходы цвета/прозрачности + reveal (transform не анимируется — идёт за скроллом) |
| `article.client.ts` | Барабан: scroll-spy, scrub, детенты, клик по секции |

## Устройство

- **Секции** — все `##` (`<h2>`) внутри `[data-case-content-body]`. Если секций < 2 — виджет не показывается.
- **Барабан** (справа): тёмная пилюля (`--color-bg-surface`) с точками. Активная точка — 8px белая (`--color-bg-contrast`), каждая следующая на 1px меньше (`8→7→6…`, min 2px) и приглушена (`--color-bg-control`). В свёрнутом виде шаг между точками — 16px (`DOT_SPACING`, т.е. ~8px зазор). При раскрытии пилюля растягивается, а точки расходятся и выравниваются по именам.
- **Имена** (слева): активное 12px, соседние 11px, дальние 10px; выровнены вправо, кликабельны. В свёрнутом виде видны ~3 имени рядом с активным (`NAME_WINDOW`); при раскрытии — полный список секций. Зона клика больше самого текста (прозрачные паддинги сверху/снизу/слева), фон не рисуется.
- **Пиннинг у краёв:** на первой секции активный кругляш прижат к верху пилюли с отступом 8px (остальные каскадом вниз), на последней — к низу; в середине активный центрируется. Так же ведут себя имена.

## Поведение

Два состояния:

- **Свёрнуто (default):** клик по видимому имени (слева) → навигация к секции без раскрытия; клик по пилюле (справа) → раскрывает меню.
- **Раскрыто:** клик по любому пункту (имя или точка) → навигация, **меню остаётся открытым**; свернуть можно тапом по пустой области пилюли, кликом вне виджета или `Esc`.

- **Scroll-spy:** позиция барабана — непрерывный дробный индекс секции от скролла (линия чтения — 35% вьюпорта). У нижнего края страницы позиция принудительно фиксируется на последней секции (иначе её заголовок не доезжает до линии чтения и она не «загорается»).
- **Плавность:** позиция интерполируется покадрово (lerp к цели, фактор `SMOOTH`); раскрытие/закрытие — тайм-based tween (`EXPAND_DURATION` + `easeOutExpo`). `prefers-reduced-motion` → без сглаживания и мгновенное раскрытие.
- **Скролл контента:** **перетаскивание** барабана вверх/вниз (порог `DRAG_THRESHOLD`, чтобы отличать клик от драга) и **колесо** над виджетом проматывают страницу напрямую; клик по секции → плавный скролл (`prefers-reduced-motion` → instant).
- **Hover:** неактивные (приглушённые) имена секций подсвечиваются до полной непрозрачности, неактивные точки — до активного (белого) цвета.
- **Детент:** лёгкое вибро при пересечении секции во время перетаскивания (`FeedbackBus`, только haptic). Клик — `tap`/`hover` через `data-feedback`.
- **Mobile (< 640px):** скрыт.
- Пере-инициализация на `astro:page-load` (View Transitions): старые слушатели снимаются через `AbortController`, `ResizeObserver` пере-меряет офсеты при изменении высоты тела.

Геометрия (размеры/шаги/сглаживание) — константы в начале `article.client.ts`.
